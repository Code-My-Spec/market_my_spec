## dcefc77 - Wire ClientUtils.CloudflareTunnel for dev.marketmyspec.com

Date: 2026-05-16 09:53:03 -0400
Author: John Davenport

Phoenix dev server now publishes through a Cloudflare named tunnel
when CLOUDFLARE_TUNNEL_ID/SECRET (and the existing CLOUDFLARE_ACCOUNT_ID)
are present in envs/dev.env. Lets the laptop dev server take public
HTTPS traffic at https://dev.marketmyspec.com — useful for OAuth
callback testing, remote QA, mobile previews, sharing WIP with
collaborators without a full UAT deploy.

**Plumbing**

- `config/runtime.exs` — in :dev only, reads the three tunnel env vars
  and writes them to `:cloudflare_tunnel` app env if all present. Missing
  vars disable the tunnel silently (cloudflared not required for normal
  dev). When wired, also overrides `:apex_host` to `dev.marketmyspec.com`
  so the AgencyHost plug treats it as the apex — without this, the plug
  parses "dev" as an agency subdomain and 302s every request to
  https://marketmyspec.com (same gotcha 8694071 fixed for UAT).

- `lib/market_my_spec_web/application.ex` — `cloudflare_tunnel_child/0`
  returns `[]` when no `:cloudflare_tunnel` config is set (test, prod,
  dev without creds), or `[{ClientUtils.CloudflareTunnel, opts}]` when
  it is. Appended after the Endpoint so the local origin is already
  listening when cloudflared connects.

**One-time setup (already done locally for this laptop)**

- `cloudflared tunnel create market-my-spec-dev` (uuid 6d58f8d8-...)
- DNS CNAME `dev.marketmyspec.com` → `<uuid>.cfargotunnel.com`,
