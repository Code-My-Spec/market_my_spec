## b60d328 - Expose engagement venue + saved-search admin to the agent via MCP

Date: 2026-05-15 14:11:16 -0400
Author: John Davenport

Stories 708 and 710 shipped with the LiveView admin and the underlying
Engagements context, but the MCP tool surface they call out in their
acceptance criteria was never built — the agent had no way to manage
venues or saved searches without going through the UI. Smoke test
this morning confirmed the gap: tools/list returned 8 tools, none of
them venue/search related.

This commit closes the gap in one shot.

**9 new MCP tool modules** under lib/market_my_spec/mcp_servers/engagements/tools/:
- add_venue, list_venues, update_venue, remove_venue (story 708)
- create_search, list_searches, run_search, update_search, delete_search
  (story 710)

Each follows the SearchEngagements pattern: use Anubis.Server.Component,
schema do/end for JSON Schema, execute/2 returning
{:reply, Response.tool() |> Response.text(Jason.encode\!(payload)), frame}.
All scope through frame.assigns.current_scope and delegate to the public
MarketMySpec.Engagements context — no direct repository imports from
the MCP layer. RunSearch returns the same {candidates, failures}
envelope as the ad-hoc search_engagements tool so the agent can treat
both surfaces interchangeably.

All 9 tools registered on MarketingStrategyServer; tools/list now
serves 17 tools over the wire (smoke-tested with curl + bearer:
add_venue → create_search → run_search → list_searches → delete_search
→ remove_venue all return clean responses with valid envelopes).

**Beefed up 9 anemic spex** per feedback_beef_up_anemic_spex.md.
Story 708's MCP criterion spex (6138, 6150-6153) were pattern-matching
on {:scaffold, :not_yet_implemented} as PASS via try/rescue, so the
QA roll-forward greened them without ever requiring the tool modules
to exist. Story 710's MCP criterion spex (6232, 6234, 6235, 6236)
were testing SavedSearchesRepository directly instead of the agent
surface their gherkin called for. Both classes now use the Anubis
testing recipe — direct alias, .execute/2, assert {:reply, %Response{},
_frame}, decode the JSON payload, then verify against the database or
the response envelope.

**Sidebar nav** in layouts.ex: Venues + Saved searches links between
Accounts and Integrations, gated on @current_scope.active_account_id
so they only render when the user has an active account.

Refs: stories 708, 710. Spex: 259/259 green. Non-spex: 168/168 green.
Live MCP smoke: 17 tools served, 6/6 round-trip calls passed.
