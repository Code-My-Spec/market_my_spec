## aeca446 - Hetzner deploy scaffolding — Dockerfile, compose, release, deploy scripts

Date: 2026-05-04 22:50:07 -0400
Author: John Davenport

Modeled after the metric_flow + fuellytics templates John pointed at.
Single-host pattern: each env (prod, uat) lives on its own Hetzner host
that already runs an `infra-postgres-1` container plus the shared
`caddy_proxy` and `db_net` external networks.

Added:
- Dockerfile — multi-stage Phoenix release. Build stage compiles in
  MIX_ENV=prod, runs assets.deploy, mix release. Includes the Dotenvy
  envs/ guard (mkdir -p envs && touch placeholders) so source\!
  doesn't crash on missing files in the container. Runner stage is
  bookworm-slim with libstdc++6 / openssl / locales / curl.
- docker-compose.yml + docker-compose.prod.yml — UAT and prod app
  stacks. No db service; DATABASE_URL points at infra-postgres-1 by
  Docker DNS through the db_net external network. Joins caddy_proxy
  for ingress. Healthcheck hits /.well-known/oauth-authorization-server
  (200 means the Endpoint is up + routing works).
- Caddyfile — reverse_proxy snippet for app.marketmyspec.com (prod);
  uat.marketmyspec.com commented out for the UAT host's Caddy.
- lib/market_my_spec/release.ex — MarketMySpec.Release.migrate/0 and
  rollback/2, called via /app/bin/migrate after each deploy.
- rel/overlays/bin/migrate + rel/overlays/bin/server — Phoenix release
  scripts (migrate evals MarketMySpec.Release.migrate; server starts
  the release with PHX_SERVER=true).
- mix.exs releases block — packages the overlays into _build/<env>/rel.
- scripts/deploy and scripts/deploy-uat — rsync + docker compose up + bin/migrate.
  Hardcoded server placeholders (<HETZNER_PROD_IP>, <HETZNER_UAT_IP>) for
  the operator to fill in.
- envs/{prod,uat}.env.example — full set: PHX_HOST, SECRET_KEY_BASE,
  POSTGRES_PASSWORD, POSTGRES_DB, CLOAK_KEY, AWS_*, OAuth creds,
  RESEND_API_KEY. DATABASE_URL is built from POSTGRES_* in the compose
  file so the env file stays minimal.
