# 2026-05-05

The day belonged to deployment infrastructure, and it fought back the whole way.

It opened quietly in the late afternoon with a dependency bump: client_utils jumped to 0.1.18, dropping a duplicate `Mix.Tasks.Compile.Spex` compiler that had been colliding with sexy_spex and breaking `mix release`. A small change, but a load-bearing one — without a working release build, nothing downstream would matter.

Then came the main event. Just after 11pm, John pushed the migration from ad-hoc deploy scripts to Kamal driven by GitHub Actions. A new GHA workflow began building amd64 images on every push to master and pushing them to GHCR. Fresh `config/deploy.yml` and `config/deploy.uat.yml` files set up Kamal with kamal-proxy disabled, keeping Caddy as the public-traffic front door via the `caddy_proxy` network. `.kamal/secrets` was wired to pull from the calling process's env, with `scripts/deploy` and `scripts/deploy-uat` populating it from SSM Parameter Store via `render-env` plus a `gh auth` token, then invoking `kamal deploy --skip-push --version=<sha>`. A post-deploy hook attached the new container to `db_net` so it could reach Postgres, and docker-compose files were switched over to DATABASE_URL passthrough to align with the per-app DB user pattern.

The next forty minutes were spent chasing the inevitable rough edges. First, Kamal's pre-deploy verification demanded a `service=market-my-spec` label on the image, which the GHA workflow hadn't been applying — a two-line fix to the build workflow. Then the post-deploy hook revealed a Kamal quirk: `--network kamal ... --network caddy_proxy --network-alias X` applies the alias to the first network listed, not to `caddy_proxy` as intended. The hook was rewritten to disconnect and reconnect `caddy_proxy` with the correct alias alongside the existing db_net attach. Finally, `bin/migrate` was failing because `readlink -f` returns the file path, not its parent, and `cd $0/../..` was navigating up from a file rather than a directory; wrapping in `dirname` set the target correctly to `/app`.

A focused infra sprint: one big architectural move, three small fixes shaking out the consequences, and a clean Kamal pipeline by midnight.
