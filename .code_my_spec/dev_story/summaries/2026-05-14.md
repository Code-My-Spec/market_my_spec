# 2026-05-14

May 14 was a marathon sprint that took the Engagements feature from blank architecture docs all the way through implementation, QA, and a fiery UAT deploy in a single day. Nine commits spanning twelve hours opened with planning paperwork and closed near midnight with two deploy-script papercuts patched after a 500-storm on UAT.

The morning began with research and design groundwork. John captured ADRs for Reddit script-app OAuth, Discourse anonymous reads, Req as the new HTTP client, and ReqCassette for testing, alongside knowledge docs, integration verify scripts, and per-component spec files for the new Engagements context. With the proposal in place, the centerpiece commit dropped: a sweeping Engagements implementation covering stories 705/706/707/708. It introduced Venue, Thread, and Touchpoint schemas, account-scoped repositories, a Source behaviour with Reddit and ElixirForum adapters, a parallel Search orchestrator using Task.async_stream, and a Posting helper that bakes UTM tracking into staged drafts. Three MCP tools and four LiveView surfaces were registered, backed by 75 BDD spex files. Earlier in the day, a security fix for story 696 closed an invitation-acceptance bug that let a signed-in user silently join the wrong account; the fix landed with a beefed-up spex case that previously passed vacuously.

The midday push then layered on QA infrastructure: a seed-port fix to stop hardcoding localhost:4008, fourteen story result_complete artifacts with screenshot trails, and a fresh EngagementsFixtures module to unbreak the spex suite. By evening, story 710 (saved searches) shipped on top of the engagements substrate with its own schema, repository, LiveView admin, and ten BDD spex.

The day ended in firefighting mode. A first UAT deploy died because HostResolver had baked marketmyspec.com at compile time, redirecting everything from uat.marketmyspec.com back to prod; the fix moved the lookup to runtime. Then a transient tailwind 504 and a forgotten migrate-before-swap step prompted Dockerfile retry loops and pre-swap migrations in both deploy scripts.
