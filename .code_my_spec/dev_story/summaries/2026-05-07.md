# 2026-05-07

The day opened with infrastructure brinkmanship. John pushed UAT off Caddy and onto kamal-proxy at 10:31, then spent the next two hours fighting Kamal's config merge semantics: a role-level `proxy: false` from `deploy.yml` kept bleeding into UAT, disabling kamal-proxy after every cutover. After a revert-reapply-revert dance, the real culprit surfaced - Phoenix's `Plug.SSL` was 301-redirecting kamal-proxy's internal `/up` healthcheck because the probe doesn't carry `X-Forwarded-Proto`. He rolled UAT back to the Caddy-fronted config, added `paths: ["/up"]` to `force_ssl.exclude`, wired a short-circuit `/up` plug into the endpoint, and pre-attached `db_net` at container-create time so the app could reach Postgres before becoming healthy. With those fixes in hand the prod cutover landed cleanly, the now-obsolete caddy_proxy reconnect post-deploy hook was deleted, and CI moved from a single QEMU-emulated buildx job to a native ARM matrix - cutting Phoenix/Elixir build times from 25-30 minutes down to 5-10.

The afternoon turned to secrets hygiene. Mirroring metric_flow's pattern, John introduced `MarketMySpec.Secrets` to fetch app secrets from SSM at boot via paged `GetParametersByPath`, leaving Kamal carrying only AWS bootstrap creds plus `APP_ENV`. That closed an operator-side render-env exfiltration vector.

Evening was a feature sprint. Story 672/673 (OAuth sign-up via Google and GitHub) finally resolved sign-ins by `(provider, provider_user_id)` instead of email, fixing a duplicate-user bug when a Google primary email changed. He ported ReqCassette-based OAuth infrastructure from code_my_spec so all eight spex could replay callbacks offline, fixed a wrong-route bug in two denial-recovery specs, and added the long-missing Google and GitHub buttons to `/users/register`. Then came story 684: a side-by-side files explorer (`FilesLive.Browser`) with an MDEx-rendered markdown pane, real account-switching via `User.active_account_id`, and a clean split of `Layouts.app` (authenticated drawer) from `Layouts.marketing` (public center-column). Seven BDD specs and a QA seed shipped alongside. A late URL-encoding bug in the tree links - `~p"/files/#{path}"` was escaping slashes and tripping `Plug.Static`'s path validator - got squashed with the splat-route interpolation form. A small `McpAuth.ConnectionInfo` helper rounded out the day.
