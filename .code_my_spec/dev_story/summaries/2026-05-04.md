# 2026-05-04

May 4 was the day Market My Spec stopped being a local project and started being a deployable one. Six commits, all stacked late into the evening, carried the app from "compiles on my machine" through "ships to Hetzner against a real DNS zone."

The day opened with a landmark commit landing the entire web and domain surface area in a single push: an Anubis-backed MCP server mounted under `/mcp`, an `McpAuth` context wrapping `ex_oauth2_provider` for RFC 9728/8414 metadata and dynamic client registration, a multi-tenant accounts model with individual-vs-agency types and `AgencyClientGrants`, a Files context with Memory/Disk/S3 adapters, and a full agency dashboard with client-switching. The brand system got applied across the board, native `data-confirm` dialogs were swapped for a testable `ConfirmModal`, Wallaby and `FeatureCase` were scaffolded, and dozens of anemic spex were rewritten into behavioral `Frame.execute` and Anubis `Response` assertions. Four migrations, hundreds of QA screenshots, and a journey plan covering five end-to-end flows shipped alongside.

From there the work pivoted to infrastructure. The Files S3 backend was rewritten from hand-rolled Req+sigv4 onto `ex_aws_s3` Operations, idempotent shell scripts for AWS bucket and IAM bootstrap arrived with an operator runbook, and five Wallaby journey tests were added under a `:wallaby` tag. A runtime `base_url` fix sorted out a long-standing bug where the install command and server URL were hardcoded to `localhost:4000` instead of reading the Endpoint's actual binding.

A short cleanup pass followed: `Endpoint.url/0` replaced the boot-time config read in `HomeLive` and `McpSetupLive`, the now-redundant `McpAuth.ConnectionInfo` module and its tests were deleted, and two bugs in the AWS bootstrap script (us-east-1's `LocationConstraint` quirk, misplaced `ExpiredObjectDeleteMarker`) were patched after a real run against AWS account 889081505590 surfaced them.

The deploy story then crystallized. A Fly.io-flavored runbook was retargeted to Hetzner Docker Compose per the existing ADR. A full deploy scaffold landed: multi-stage Dockerfile, prod and UAT compose files joining the shared `caddy_proxy` and `db_net` networks, `MarketMySpec.Release` with migrate and rollback, overlay scripts, and rsync-based deploy commands. Finally, the placeholder IPs were swapped for the actual Hetzner hosts (`178.156.143.212` ash, `46.225.105.88` nbg1), Caddy was pointed at the apex `marketmyspec.com`, and Cloudflare DNS records were cut over via the API.

A sprint, top to bottom, ending with a project that knows where it lives.
