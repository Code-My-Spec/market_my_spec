## Issues

### /mcp-setup route is not registered in the router

#### Severity
CRITICAL

#### Scope
APP

#### Description
`MarketMySpecWeb.McpSetupLive` exists at `lib/market_my_spec_web/live/mcp_setup_live.ex` but no `live "/mcp-setup"` route is registered in `lib/market_my_spec_web/router.ex`. Navigating to `http://localhost:4007/mcp-setup` returns a Phoenix `NoRouteError` (HTTP 404) for all visitors, authenticated or not.

Required fix: add the route to the `:require_authenticated_user` live_session in the router, e.g.:

```elixir
live "/mcp-setup", McpSetupLive, :index
```

Reproduced: `curl -sS -I http://127.0.0.1:4007/mcp-setup` returns `404 Not Found`.

### McpSetupLive template missing required data-test attributes

#### Severity
HIGH

#### Scope
APP

#### Description
The BDD specs (criterion 5695, 5696) assert the presence of `[data-test='server-url']`, `[data-test='install-command']`, and `[data-test='oauth-instructions']` elements. None of these attributes exist in the current template at `lib/market_my_spec_web/live/mcp_setup_live.ex`.

The template renders `{@server_url}` inside a `<pre class="mockup-code"><code>` block and `{@install_command}` similarly, but neither `<pre>`, `<code>`, nor any parent element carries a `data-test` attribute.

Additionally, there is no element with `data-test="oauth-instructions"` — the OAuth sign-in step is a plain `<li>` with descriptive text but no test hook.

Required fix: add `data-test` attributes to the relevant elements so the BDD specs and future automated tests can target them.

### McpSetupLive uses Endpoint.url() instead of McpAuth.ConnectionInfo

#### Severity
MEDIUM

#### Scope
APP

#### Description
The spec at `.code_my_spec/spec/market_my_spec_web/mcp_setup_live.spec.md` states the LiveView "Pulls server URL and OAuth flow details from McpAuth.ConnectionInfo." The current implementation in `mount/3` calls `MarketMySpecWeb.Endpoint.url() <> "/mcp"` directly instead of delegating to `MarketMySpec.McpAuth.ConnectionInfo.server_url/0` or `setup_info/0`.

The `MarketMySpec.McpAuth.ConnectionInfo` module (`lib/market_my_spec/mcp_auth/connection_info.ex`) exists and exposes `server_url/0`, `install_command/0`, and `setup_info/0` — the LiveView should use these instead of computing the URL inline. This also means the install command rendered on the page (`claude mcp add market-my-spec`) is missing the URL argument that `ConnectionInfo.install_command/0` includes (`claude mcp add market-my-spec <server_url>`).
