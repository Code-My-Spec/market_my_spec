### Thread and Venue routes missing from running dev server

#### Severity
HIGH

#### Scope
APP

#### Description
The routes `GET /accounts/:id/threads`, `GET /accounts/:account_id/threads/:thread_id`, and `GET /accounts/:id/venues` are defined in `lib/market_my_spec_web/router.ex` but return 404 from the running dev server.

Root cause: the dev server was started before these routes were added to `router.ex`. Phoenix's code reloader does not automatically recompile route tables on router changes — a full server restart is required.

To reproduce: navigate to `http://localhost:4008/accounts/<any-account-id>/threads` while the server is running with stale router code.

To fix: restart the dev server with `PORT=4008 mix phx.server`. After restart, the routes should return 200.

This blocks UI testing of ThreadLive.Index and ThreadLive.Show. All BDD spex tests pass because they test the Elixir module layer directly and do not require HTTP routes.
