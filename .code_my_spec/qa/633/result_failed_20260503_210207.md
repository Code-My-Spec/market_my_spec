fail

## Scenarios

### 1. Page loads and renders

pass

Navigated to `http://localhost:4007/`. Server returned HTTP 200. The page rendered without errors. However, the page served is the default Phoenix 1.8 scaffold (PageController `:home` action), not the custom `MarketMySpecWeb.HomeLive`.

Evidence: `.code_my_spec/qa/633/screenshots/633-initial-load.png`

### 2. Artifact preview in the hero (Criterion 5665 / 5666)

fail

No element with `data-test="artifact-preview"` exists on the page. The page is the Phoenix scaffold, not HomeLive. No strategy artifact, positioning content, or hero section matching the spec is rendered. Both criteria 5665 and 5666 fail.

### 3. Hero headline present (Criterion 5665)

fail

No element with `data-test="hero-headline"` exists on the page.

### 4. Install command accessible without auth gate (Criterion 5667 / 5668)

fail

No element with `data-test="install-command"` exists on the page. The text `claude mcp add` does not appear anywhere on the page. No copy button (`data-test="copy-button"`) is present. Both criteria 5667 and 5668 fail.

### 5. BYO-Claude benefit line below the hero (Criterion 5669 / 5670)

fail

No element with `data-test="byo-claude-benefit"` exists on the page. Neither "bring your own claude" nor "don't markup your tokens" appears in the page text. Both criteria 5669 and 5670 fail.

### 6. Messaging-guide phrase audit (Criterion 5671 / 5672)

fail

The canonical positioning line "Marketing for founders, in Claude Code" is absent from the page. The page does not contain any of the banned phrases, but it also lacks all of the required content. Criterion 5671 fails because the required canonical line is missing. Criterion 5672 (ban check) cannot be meaningfully passed because the implementation is absent.

### 7. Agency CTA below install (Criterion 5673 / 5674)

fail

No element with `data-test="agency-cta"` exists on the page. Neither "run an agency" nor "talk to john" appears in the page text. Both criteria 5673 and 5674 fail.

### 8. Router wiring

fail

The root cause of all failures: the router routes `GET /` to `PageController, :home` (the default Phoenix scaffold action), not to `MarketMySpecWeb.HomeLive`. The `HomeLive` module exists at `lib/market_my_spec_web/live/home_live.ex` but is not mounted in the router. Additionally, even if the router were corrected, `HomeLive` itself is missing all required `data-test` attributes, the install command, the artifact preview, the BYO-Claude benefit line, the agency CTA, and the canonical positioning line "Marketing for founders, in Claude Code".

## Issues

### Router does not mount HomeLive at the root route

#### Severity
CRITICAL

#### Scope
APP

#### Description
The router at `lib/market_my_spec_web/router.ex` line 23 routes `GET /` to `PageController, :home`, which serves the default Phoenix framework scaffold page. `MarketMySpecWeb.HomeLive` exists at `lib/market_my_spec_web/live/home_live.ex` but is not wired into the router. Every criterion in story 633 fails as a result.

Fix: replace `get "/", PageController, :home` in the public browser scope with a LiveView mount, e.g. add `live "/", HomeLive, :index` inside the `:current_user` live_session, or a separate public live_session for the root route.

### HomeLive missing all required data-test attributes and content

#### Severity
HIGH

#### Scope
APP

#### Description
Even if the router is fixed, the current `HomeLive` implementation at `lib/market_my_spec_web/live/home_live.ex` does not satisfy the story's acceptance criteria. Specifically missing:

- `data-test="artifact-preview"` — no strategy artifact is rendered in the hero
- `data-test="hero-headline"` — the `<h1>` has no data-test attribute
- `data-test="install-command"` — no install command block with `claude mcp add` is present
- `data-test="copy-button"` — no copy affordance exists
- `data-test="byo-claude-benefit"` — the card titled "Bring your own Claude" lacks this attribute and does not contain required copy ("don't markup your tokens" pattern)
- `data-test="agency-cta"` — no agency CTA with "run an agency" / "talk to john" copy exists
- Canonical positioning line "Marketing for founders, in Claude Code" — not present anywhere on the page
- The hero copy ("Marketing strategy your agent runs.") does not match the spec's required headline pattern

The BYO-Claude card exists as a feature card in a 3-column grid but does not have the required data-test attribute and contains "No token markup. Connect your own Claude Code subscription. We never resell tokens." — the spec requires "don't markup your tokens" (case-insensitive match).
